## ЧАСТИНА 1 — ТЕХНІЧНИЙ СТЕК

### Фронтенд
- **Framework:** Angular 16.2.12 (виробничий збірник webpack з content-hash файлами)
- **UI Kit:** Angular Material (компоненти `mat-*`)
- **Шрифт:** Montserrat sans-serif (Google Fonts)
- **Іконки:** Stockholm Icons (SVG sprites) + Google Material Icons
- **Версія застосунку:** 4.0.136 (відображається в підвалі кожної сторінки)
- **Маршрутизатор:** Hash Router (усі URL мають префікс `#/`)
- **Мова UI:** Українська

### Бекенд та інфраструктура
- **CDN / WAF:** Cloudflare (HTTP/3, zstd compression, Cloudflare Insights analytics)
- **Протокол:** HTTPS + HTTP/3 (QUIC)
- **Стиснення:** zstd (Cloudflare Zstandard)
- **Мікросервіси:** 2 API-сервіси
  - `/api/tracker/` — основна CRM-логіка
  - `/api/injector/` — UAD (User Action Director), автоматизація сценаріїв
  - `/api/auth/` — автентифікація та профіль
  - `/api/geo/` — географічні довідники
- **SEO:** `noindex, nofollow` (закрито від індексації)
- **Публічна документація:** ВІДСУТНЯ (swagger/redoc/openapi.json — 404)

### Автентифікація
- **Тип:** Session UUID Bearer Token (НЕ JWT — немає крапок)
- **Зберігання:** `localStorage`, ключ `InNlc3Npb25LZXki` (base64 від `"sessionKey"`)
- **Декодування:** `JSON.parse(atob(localStorage.getItem('InNlc3Npb25LZXki')))` → UUID рядок
- **Заголовок:** `Authorization: Bearer <uuid>`
- **Активна компанія:** `localStorage` ключ `ImFjdGl2ZUNvbXBhbnlJZCI=` (base64 від `"activeCompanyId"`)
- **Поведінка сесії:** Закінчується під час роботи без видимого механізму refresh-токену. Angular перехоплює 401, виконує re-auth та повторює запит.

## ЧАСТИНА 2 — API-АРХІТЕКТУРА

### Конвенції API
```
Пагінація:   ?skip=0&take=10
Сортування:  ?sort[fieldName]=asc|desc
Фільтрація:  ?filter[fieldName][gte]=value&filter[fieldName][lte]=value
Список (dropdown): /api/tracker/{entity}/list  (без пагінації)
Список (таблиця):  /api/tracker/{entity}?skip=0&take=10
```

### Префікси кодів об'єктів
| Префікс | Об'єкт |

| `FP-`   | Fraud Profile (Фрод профіль) |

## ЧАСТИНА 5 — КОНКУРЕНТНИЙ АНАЛІЗ

### Сильні сторони HyperOne
1. **Дворівнева маршрутизація** (Affiliate → Hub → Stream → Brand) — більш гнучка ніж у більшості конкурентів
2. **Автологін** — автоматизований редирект ліда в бренд з відстеженням кожного кроку
3. **UAD менеджер** — сценарії автоматизації (відсутній у дешевших CRM)
4. **134 готових інтеграції** — широке покриття брокерів
5. **Подвійна фрод-система** — бінарні профілі + числова ML-оцінка
6. **17 типів Telegram-сповіщень** — детальний моніторинг
7. **Резервний режим хаба** — автоматичний failover при невдалій відправці
8. **Конфігурований холд конверсій** — захист від стягнення комісій до підтвердження
9. **27 аналітичних вимірів** — глибока аналітика

### Слабкі сторони / UX-болі
1. Відсутня публічна документація API (нема Swagger/OpenAPI)
2. Сесії закінчуються непередбачувано без refresh-токенів
3. Лише 1 вертикаль (Base) — обмежена мультивертикальність
4. Не адаптовано під мобільні пристрої
5. Відсутнє темне/світле перемикання теми (тільки темна)
6. Навігація по 14 модулях потребує прокрутки в бічній панелі
7. Форма створення ліда дуже довга (не розбита на кроки)
