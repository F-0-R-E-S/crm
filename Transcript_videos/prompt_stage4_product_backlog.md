Ты — Senior Product Manager с опытом в B2B SaaS для финтех/affiliate-маркетинга.

Твоя задача — создать полный структурированный продуктовый бэклог для нового

═══════════════════════════════════════════════
КОНТЕКСТ: ЧТО МЫ СТРОИМ
═══════════════════════════════════════════════

Продукт: CRM-платформа для дистрибуции лидов в крипто/форекс нише.

Целевые пользователи:
  - Buying networks (покупающие сети): команды медиабайеров, 5-50+ человек
  - Affiliate networks: агрегаторы трафика, работающие с 10-200+ аффилейтами
  - Media buyers: одиночки и небольшие команды (3-10 байеров)

Основные сценарии использования:
  1. Аффилейт отправляет лид через API → система маршрутизирует к нужному брокеру
  2. Система проверяет лид на фрод перед отправкой брокеру
  3. Брокер обновляет статус лида → система уведомляет аффилейта
  4. Команда смотрит аналитику: конверсии, депозиты, P&L по аффилейтам/брокерам
  5. Менеджер переотправляет отказные лиды другим брокерам по расписанию

Изучены 4 конкурента. Их сильные и слабые стороны:

HYPERNET (HyperOne) — $499-$1499/мес — Overall 8.1/10

  Слабо: аналитика (6) — нет time-series, нет P&L, нет drill-down
         UX (7) — нет мобайла, сложный онбординг, перегруженный интерфейс
         Multi-tenancy (7) — неясная изоляция между компаниями

LEADGREED — Overall 8.0/10
  Сильно: SLOTS vs CHANCE алгоритмы, визуальный редактор дистрибуций,
          детальная аналитика с drill-down, fake FTD, статус-override,

CRM MATE — $700/мес (один тариф)

Наши конкурентные преимущества (то что мы должны делать лучше всех):

2. UX И ОНБОРДИНГ
   Конкуренты сложны в настройке (HyperOne — высокая onboarding complexity).

   Нет setup fee. Публичные цены. Freemium или trial. Нет cap'ов на fraud checks.

5. МОБИЛЬНЫЙ МОНИТОРИНГ
   Ни один конкурент не имеет мобильного интерфейса.
   Мы даём нативный mobile dashboard для байеров.

СПЕЦИАЛЬНЫЕ ИНСТРУКЦИИ ПО КОНТЕНТУ
═══════════════════════════════════════════════

ВАЖНО при генерации Acceptance Criteria:
  - Указывай конкретные числа: не "быстро", а "< 500ms"
  - Указывай edge cases: что происходит при ошибке, при пустых данных
  - Указывай безопасность: авторизация, валидация, rate limiting
  - Для API: метод, URL, request/response схема, коды ошибок

ВАЖНО для Epic-02 (Routing):
  Реализуй ОБА алгоритма:
  - WEIGHTED ROUND-ROBIN (как HyperOne) — для равномерной дистрибуции
  - SLOTS vs CHANCE (как Leadgreed) — для статистически корректной выборки
  Дай пользователю выбор алгоритма per-flow.

ВАЖНО для Epic-10 (Analytics):
  Это наше главное преимущество. Сделай stories для:
  - Time-series графики с drill-down (которых НЕТ у конкурентов)
  - Affiliate-level P&L dashboard
  - Hub/broker ROI comparison
  - Traffic quality cohort analysis
  - Shave detection analytics
  - Predictive cap exhaustion warnings

ВАЖНО для Epic-07 (Anti-Fraud):
  Учти что у HyperOne лимиты на fraud checks по тарифу (100/2000/4000).
  Мы предлагаем: unlimited fraud checks в базовом тарифе как дифференциатор.
  Stories должны отражать это позиционирование.

ВАЖНО для Epic-13 (Onboarding):
  Цель: от регистрации до первого отправленного лида за < 30 минут.
  HyperOne требует: создать вертикаль → аффилейта → бренд → интеграцию →
  хаб → поток → тест лид = 7+ шагов без помощи.
  Мы делаем: wizard с шаблонами, каждый шаг с объяснением и примером.
